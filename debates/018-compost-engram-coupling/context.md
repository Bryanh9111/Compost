# Debate 018: Compost ↔ Engram 关系定位 (peer + opt-in 桥接原则)

**Date**: 2026-04-16
**Rounds**: 1 (thorough, 500 words/advisor)
**Mode**: cross-critique

## 触发
用户战略校准:
> "compost 最终是个可自我进化学习的大脑, engram 和 compost 不一定要深度依赖, 两者可以有不同 layer 的重心或者交流, 但不要让一边离不开另一边"

## 背景
- debate 016 技术层已解耦: 独立 compost_cache 表 + 独立 MCP tool + 默认 recall 不含
- Opus 写的 v2 最终形态记忆里 "桥接" 位置太突出 (5 大应用场景之一, 用"killer"形容)
- 问题: 技术对齐, 叙事错位

## 6 判决点
1. **"独立自进化大脑"过度包装?** Compost 离开所有 agent 客户, 还算自进化吗? 本质是否只是被动 ingest + 主动 reflect, 没有"自主学习"野心?
2. **两者协同边际价值多大?** 场景 C (两装+opt-in 桥接) 相对 A∪B 的增量?
3. **"不让一边离不开另一边"矛盾吗?** 真正对称独立还是非对称依赖?
4. **叙事语言规范?** Compost-为-Engram / Engram-消费-Compost / 中立互操作?
5. **compiled origin 真实命运?** 默认关闭 → 稀疏数据 → 值得保留吗?
6. **v2 记忆要不要立刻 supersede v3?** 还是按纪律等 Session 3 后固化?

## 4 方案
- **A**: 维持现状 (技术保留 + 叙事修正 + 记忆暂缓)
- **B**: 彻底砍 v3.5 桥接 (纯独立)
- **C**: 承认非对称依赖 (Engram 可选消费 Compost, 反向无)
- **D**: 桥接迁第三方 plugin (engram-compost-adapter)
