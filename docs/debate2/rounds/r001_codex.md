# Codex — D3+Policy

Stay hybrid. Exact boundary: Node/Bun owns `packages/compost-core/**`, `packages/mcp-server/**`, SQLite/LanceDB access, scheduler, adapters, and calls `compost-ingest` as a subprocess over JSON stdin/stdout; Python owns only `tools/compost-ingest/**` and returns normalized `Document[]`, spans, and extractor metadata, never DB writes. Gemini’s weakest point is treating distribution as the strategy when maintainability is the actual constraint: a single `bun` install that silently downgrades extraction quality is operationally worse than `install.sh && uv sync`. Phase 1 pure-Node failure mode: a mixed PDF with tables plus scanned pages goes through `pdf.js` text extraction plus ad hoc OCR, loses table structure and formulas, and writes bad chunks into `observations` and `chunks`; then `SELECT transform_policy, source_uri FROM observations WHERE source_uri LIKE '%.pdf';` reports “successful” ingests whose downstream facts are garbage. Today there is no Node-equivalent stack to `docling` + `unstructured` + `trafilatura` for that workload next week.

Use date-stamp with revision counter: `tp-2026-04-02`. Semver is fake precision here because “backward compatible policy” is meaningless once chunking, prompts, or dedup thresholds change; replay semantics are always policy-specific. Git SHA is reproducible but hostile in daily ops and code review. The durable pattern is: immutable registry entry in `packages/compost-core/src/policies/registry.ts`, optional per-policy snapshot file `packages/compost-core/src/policies/tp-2026-04-02.json`, and DB rows keyed by the readable id: `observations.transform_policy = 'tp-2026-04-02'`. Three years from now, diffs stay obvious, replay stays deterministic, and SQL/debugging stays humane.
